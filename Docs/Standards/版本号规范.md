## OpenXRRT 版本号规范

### 1. 适用范围
- 本规范用于 `module/OpenXRRT/src/State/RuntimeState.h` 中的 `RuntimeState::kRuntimeCodeVersion`。

### 2. 格式定义
- 固定格式：`MAJOR.MINOR.PATCH+YYMMDD.N`
- 示例：`0.0.1+260319.2`

字段说明：
- `MAJOR.MINOR.PATCH`：语义化版本（SemVer）。
- `YYMMDD`：日期，按北京时间（Asia/Shanghai）计算。
- `N`：当日递增序号，从 `1` 开始。

### 3. 递增规则
- 每次修改 OpenXRRT 代码后，必须递增版本号。
- 同一天内连续改动：
  - 至少递增 `N`（`N + 1`）。
  - 若行为变化需要反映发布语义，则同时调整 `MAJOR/MINOR/PATCH`。
- 日期变化后：`YYMMDD` 更新为新日期，`N` 重置为 `1`。
- SemVer 调整建议：
  - 兼容修复：`PATCH + 1`
  - 兼容功能新增：`MINOR + 1`，`PATCH` 归零
  - 不兼容变更：`MAJOR + 1`，`MINOR/PATCH` 归零

### 4. 操作示例
- 当天首次修复：`0.0.1+260319.1`
- 当天第二次修复：`0.0.1+260319.2`
- 次日首次修复：`0.0.1+260320.1`
- 同天发布一个兼容新功能：`0.1.0+260319.3`

### 5. 记录位置
- 版本常量定义：`module/OpenXRRT/src/State/RuntimeState.h`
- 启动日志输出：`module/OpenXRRT/src/Utils/XrWorkflowUtils.hpp`
- Windows 资源版本：`module/OpenXRRT/src/DLL/OpenXRRT.rc`

### 6. OpenXRRT.rc 同步规则
- `OpenXRRT.rc` 必须与运行时版本同步更新。
- `FILEVERSION/PRODUCTVERSION` 使用 4 段数字格式：`MAJOR,MINOR,PATCH,MMDD`。
  - 例如运行时版本 `0.0.1+260319.3`，则资源版本数字段为 `0,0,1,319`。
- `StringFileInfo` 中 `FileVersion/ProductVersion` 使用点分格式：`MAJOR.MINOR.PATCH.MMDD`。
  - 例如：`0.0.1.0319`。
